# EMS 项目总览

## 1. 背景与目标
- 背景：独立开发者，构建一个可部署的 EMS 系统，支持设备数据采集、时序存储、实时状态、历史查询、控制下发、审计与后续告警。
- 目标：
  1) WSL2 本地可一键启动并形成闭环（采集→规范化→入库→查询；控制→回执→审计）
  2) 后端 Rust 模块化单体，边界清晰，未来可拆服务
  3) 多租户 SaaS：tenant + project 隔离从第一天生效
  4) 前端采用 pure-admin-thin，后端提供兼容登录/刷新/动态路由的契约

## 2. 非目标（明确不做）
- 不在 MVP 阶段做：多区域部署、复杂计费、全量规则引擎、全协议覆盖（优先 MQTT）
- 暂不做：K8s 生产化落地（先占位，但不投入实现）

## 3. 里程碑（可演示的增量）
- M1：后台可登录 + 鉴权 + 动态路由菜单可见（最小前端联调）
- M2：项目内元数据 CRUD（gateway/device/point/point_source）
- M3：MQTT 采集闭环（写入 Timescale + realtime last_value）
- M4：控制闭环（command 下发→回执→审计查询）
- M5：告警框架占位（规则管理 API + 事件表 + 引擎接口）

## 4. 术语表
- Tenant：租户（SaaS 组织边界）
- Project：租户下项目（现场/园区/站点边界）
- Gateway：网关/采集边界
- Device：设备
- Point：点位（可采集或可控的变量）
- RawEvent：协议输入原始事件
- PointValue：规范化后的点位值
